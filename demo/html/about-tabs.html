The tabset you are viewing is generated from an RDF document included as a web page component. You can see the complete code used to implement this in the RDF and HTML tabs. Here's an overview of what's going on.

1. We create an RDF document containing tab definitions, which, at a minimum, contain a label and a URL.

:Tabset
  schema:listuix:tabs( :Tab1 :Tab2 ).

:Tab1
  rdfs:label "Home";
  schema:url <https://example.org/home>.

:Tab2
  rdfs:label "Work";
  schema:url <https://example.org/work>.

2. We create a tabs componenet in an HTML page pointing to the RDF tabs definition as the data-source. 

<div class="solid-ui-tabs" data-source="tabs.ttl"></div>

3. We import the solid-ui and solid-ui-components libraries

<script src="../solid-ui/lib/webpack-bundle.js"></script>
<script src="./solid-ui-components.js"></script>
That's it. Once the libraries are loaded, HTML for the tabs will be automagically generated. There are three steps to A tab's component :Tabset uix:tabs( ... ).

Custom Tab Actions

A tab definition may optionally include a schema:potentialAction to specify the name of a javascript method to pre-process a tab's content. For example, this displays an HTML page as preformatted rather than as HTML. 

:Tab3
    rdfs:label "HTML Sample" ;
    schema:url <example.html> ;
    schema:potentialAction "preformat" .

The preformat action is supported by solid-ui-components. Developers will nee to define any other actions themselves.
